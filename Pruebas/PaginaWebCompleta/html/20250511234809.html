Abstract
Research soware (i.e., the tools and scripts developed as part of a research investigation) are key to
support the results described in academic publications. However, current citation practices followed by
researchers make it dicult to automatically identify and reconcile the tools used in existing publications
(dierent names are used for the same tool, dierent citing styles, etc.). In this demo we address this
issue by integrating SofCite, a state of the art named entity recognition model designed to nd soware
mentions in the biomedical domain, with Soalias-KG, a Knowledge Graph of soware aliases, in order
to reconcile soware tools in a text.

We base our work on [
3

the authors extracted soware mentions from over 3.8 million papers in PubMed Central,
6

based on Jaro-Winkler distance [

likely package managers (Pypi,

link, basic metadata of the soware mention is downloaded from the corresponding platform
(description, package URL, identier, etc.). Unfortunately, the results are stored in pickle and
csv les, designed to be used in notebooks. We have cleaned and transformed these results into
the Soalias-KG, focusing on the clustering analysis of soware mentions (i.e., soware aliases)
to facilitate reconciliation through SPARQL queries. We have also expanded the results with
soware entities from Wikidata [7], in order to generalize the application domain of the KG.
Soalias-KG includes two main types of entities: soware aliases, i.e., soware mentions as
detected in any scientic publication, and groups, which represent single soware applications
(schema:SowareApplication) grouping a cluster of aliases based on [
3
]. Groups are described
with a canonical name, based on their most frequent alias, and are described with metadata
such as URL, license, etc. All metadata are obtained from the information found in external
sources (Pypi, CRAN, etc.) and represented using Schema.org [
8
]. The metadata platform used
to enrich all groups is also kept in the KG (schema:provider), in order preserve the provenance
of each record. Groups are linked with their corresponding aliases with the alias property (a
soware group has one or more aliases).
The CZI dataset [
3
] consists on two sets of les. A rst set of pickle les describe the soware
mention groups matched to each external platform (Pypi, CRAN, etc.), together with a le
assigning an id to each soware mention in a publication. A second set of CSV les contain the
metadata for each soware entry on each external platform.

This demo integrates a state of the art named entity recognition model for detecting soware
mentions in text (SoCite) with Soalias-KG, a novel Knowledge Graph of soware aliases that
is used for tool reconciliation. Soalias-KG is based on an existing analysis in the scientic
literature over millions of paper in the biomedical domain [
3
], contains more than 50K unique
aliases (belonging to more than 30K unique tools) and is enriched with over 8K tools from
Wikidata. Thanks to our demo, we can easily navigate through existing tool aliases, as well as
detect of potential potential omissions in the NER model and our Knowledge Graph.
Our next steps will focus on identifying and addressing clustering errors (e.g., by looking
into soware groups with dierent tool URLs, joining soware groups sharing the same aliases,
etc.) and expanding Soalias- KG with Somesci KG [
13
], another KG of soware mentions of
smaller size that is already aligned with Wikidata.
