Abstract
Research soware (i.e., the tools and scripts developed as part of a research investigation) are key to
support the results described in academic publications. However, current citation practices followed by
researchers make it dicult to automatically identify and reconcile the tools used in existing publications
(dierent names are used for the same tool, dierent citing styles, etc.). In this demo we address this
issue by integrating SofCite, a state of the art named entity recognition model designed to nd soware
mentions in the biomedical domain, with Soalias-KG, a Knowledge Graph of soware aliases, in order
to reconcile soware tools in a text.

We base our work on [
3

the authors extracted soware mentions from over 3.8 million papers in PubMed Central,
6

based on Jaro-Winkler distance [

likely package managers (Pypi,
7

8
Bioconductor
9

10

repositories (GitHub
11

link, basic metadata of the soware mention is downloaded from the corresponding platform
(description, package URL, identier, etc.). Unfortunately, the results are stored in pickle and
csv les, designed to be used in notebooks. We have cleaned and transformed these results into
the Soalias-KG, focusing on the clustering analysis of soware mentions (i.e., soware aliases)
to facilitate reconciliation through SPARQL queries. We have also expanded the results with
soware entities from Wikidata [7], in order to generalize the application domain of the KG.
    
Soalias-KG includes two main types of entities: soware aliases, i.e., soware mentions as
detected in any scientic publication, and groups, which represent single soware applications
(schema:SowareApplication) grouping a cluster of aliases based on [
3
]. Groups are described
with a canonical name, based on their most frequent alias, and are described with metadata
such as URL, license, etc. All metadata are obtained from the information found in external
sources (Pypi, CRAN, etc.) and represented using Schema.org [
8
]. The metadata platform used
to enrich all groups is also kept in the KG (schema:provider), in order preserve the provenance
of each record. Groups are linked with their corresponding aliases with the alias property (a
soware group has one or more aliases).

assigning an id to each soware mention in a publication. A second set of CSV les contain the
metadata for each soware entry on each external platform.

This demo integrates a state of the art named entity recognition model for detecting soware
mentions in text (SoCite) with Soalias-KG, a novel Knowledge Graph of soware aliases that
is used for tool reconciliation. Soalias-KG is based on an existing analysis in the scientic
literature over millions of paper in the biomedical domain [
3
], contains more than 50K unique
aliases (belonging to more than 30K unique tools) and is enriched with over 8K tools from
Wikidata. Thanks to our demo, we can easily navigate through existing tool aliases, as well as
detect of potential potential omissions in the NER model and our Knowledge Graph.
Our next steps will focus on identifying and addressing clustering errors (e.g., by looking
into soware groups with dierent tool URLs, joining soware groups sharing the same aliases,
etc.) and expanding Soalias- KG with Somesci KG [
13
], another KG of soware mentions of
smaller size that is already aligned with Wikidata.
